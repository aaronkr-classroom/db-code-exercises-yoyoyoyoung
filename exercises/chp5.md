# 5장 연습문제

## 퍼즐

**가로**
1. 요구 분석 내용 가운데 ER 다이어그램이나 트랜잭션 명세서에 표현하기 어려운 운영상의 제약사항
- A. 제약조건
3. DB에서 하나의 논리적 기능을 수행하기 위한 작업의 기본 단위
- A. 트랜잭션
4. 데이터 요구 분석 명세서를 기초로 하여 DBM S오는 독립적인 개염적 데이터 모델， 즉 개념적 스키口털 표현하는 것
- A. 엔티티관계모델링
8. 개체나 관계의 특성을 나타내는 요소
- A. 속성
9. 띠터 젠 표기법에서 속성을 표헌히는 기호
- A. 원형

**세로**
2. 트랜잭션 요구 분석 명세서를 기초로 하여 업무 단위별로 유형별 트랜잭션을 표헌하는 것
- A. 트랜잭션모델
4. DB가 표현하려고 히는 유 · 무형의 정보 대상으로서 현실 세계에 존재하면서 서로 구별될 수 있는 요소
- A. 엔티티
5. 피터 젠 표기법에서 개체를 표현하는 기호
- A. 개체사각형
6. 개체 간의 의미 있는 언결 또는 언관성을 의미하는 요소
- A. 관계성
7. 피터 젠 표기법에서 관계를 표현하는 기호
- A. 마름모

## 연습문제

1. 개념적 설계 단계의 두 가지 주요 업무가 무엇인지 설명하시오.
A. 개념적 데이터 모델링, 트랜잭션 모델링 / 요구사항 분석 결과를 토대로 데이터의 구조를 표현

2. ER 모벨을 활용해서 개념적 스키마 모댈링을 수행하는 순서를 바르게 나열하시오.
  1. 개체 타입의 속성을 식별한다.
  2. 개체 타입 간의 관계 타입을 식별한다.
  3. 관계 타입의 속성을 식별한다.
  4. 관계 타입의 유형과 카디널리티(옵션)를 결정한다.
  5. 데이터 요구 분석 명세서를 기초로 하여 핵심 개체 타입을 식별한다.
  6. 개체 타입의 주 식별자， 즉 기본커 속성을 결정한다.
  7. ER 다이어그램이 데이터 요구 분석 명세서에 포함된 모든 요구사항을 반영히는지 검증한다.
  8. 식별한 개체 타입과 관계 타입， 개체 및 관계 타입의 속성을 ER 다이어그램으로 표현한다.
  A. 5 -> 1 -> 6 -> 2 -> 4 -> 3 -> 8 -> 7

3. 다음 중 개체 식별 방법에 대한 설명이 잘못된 것은?
  1. 데이터 요구 분석 명세서에서 주로 주어나 목적어로 표현된 것이 개체 후보가 된다.
  2. 개체 후보를 검토하면서 개제 이름을 변경할 수 있다.
  3. 실제 데이터 수가 2 개 이상 존재해야 최종 개체가 된다.
  4. 통일한 의미를 갖는 개체도 이름이 다르면 최종 개체가 된다.
  A. 4번

4. 디음 중 개체 이름을 결정할 때의 유의 사항이 잘못 설명된 것은?
  1. 가능하면 약어를 사용해서 간단히 표현한다
  2. 이름의 일관성을 위해 이름 부여 규칙을 미리 정하는 것이 좋다.
  3. 단수 명사를 시용한다.
  4. 현업에서 익숙한표준용어를시용한다.
  A. 1번

5. 디음 중 관계 식별과 관련된 설명이 잘못된 것은?
  1. 데이터 요구 분석 명세서에서 주로 동사로 표현된 부분을 관계로 식별한다.
  2. 식별된 개체 가운데 관계로 표현승}는 것이 적합한 것은 관계로 전환한다.
  3. 관계는개체와관련된 업무수행 규칙에 해당한다.
  4. 한 번 설정된 관계는 변경하기 어려우므로 신중히 결정해야 한다.
  A. 4번

6. 다음 중 속성 식별과 관련된 설명이 잘못된 것은?
  1. 데이터 요구 분석 명세서에서 개체나 관계를 설명히는 명사를 개체의 속성으로 식별한다.
  2. 데이터 요구 분석 명세서에서 개체의 기본키 속성을 식별할 수 없을 때는 기본카 역할을 할 속성을추가한다.
  3. 개념적 설계 단계에서는 주요 속성만 식별한다.
  4. 필요한 경우 속성 이름을 변경할 수 있다.
- A. 3번

7. (____) 안에 적합한 단어는?

> 피터 젠 표기법에서 관계는 1 개 이상의 개체를 실선으로 연결하는 (____)로 표현하고, (____)을 실선 위에 표시한다.
 A. 마름모 / 관계이름

8. 다음 중 관계 이름을 결정할 때의 유의 사항이 잘못 설명된 것은?
  1. 개체 간의 관련성이 명확하도록 표현한다.
  2. 현업에서 통용되는 전문 용어로 표현한다.
  3. 관계 이름은 DB 안에서 유일해야 한다.
  4. 가능하면 개체 간의 관련성을 나타내는 동사로 표현한다.
  A. 3번

9. 식별자에 대한설명이 잘못된 것은?
  1. 식별자는 개체가 존재히는 동안 값이 변경될 수 있다.
  2. 식별지는 반드시 유일한 값을 가져야 한다.
  3. 개체의 특성을 대표할수 있는 식별자를 기본키로 결정한다.
  4. 모든 개체는 반드시 하나 이상의 식별자를 포함해야 한다.
  A. 1번

10. 속성 이름을 결정할 때의 유의 사항이 잘못 설명된 것을 모두 고르시오.
  1. DB 안에서는 유일한 이름을 부여해야 한다.
  2. 속성 이름 앞에 개체 이름이 포함되지 않아야 한다.
  3. 현업에서 익숙한표준용어를사용한다.
  4. 유지보수를 위해 전체 이름보다는 약어를 사용한다.
  A. 1, 4번

11. 개념적 설계 단계에서 아래 ER 다이어그램을 작성한 다음에 정의한 제약사항으로 적절한 것은?
  1. 각문화센터의 센터장은 1 명이다
  2. 강좌마다 강의실이 1 개씩 배정된다.
  3. 강좌명이 통일한 강좌라도 개강 일자가 다르면 새로운 강좌로 본다.
  4. 강좌명이 동일한 강좌는 여러 번 개설될 수 있으나 한 분기에는 한 변만 개설된다.
  A. 4번

12. 트랜잭션 모넬령의 필요성에 대한 설명이 잘못된 것은?
  1. DB 설계 초기에 주요 트랜잭션의 기능적 특성을 파악할 수 있다.
  2. 추후 구현 단계에서 트랜잭션을 처리하는 응용 프로그램을 작성하는 기초 자료가 된다.
  3. 데이터의 구성과 표현을 파악할 수 있다.
  4. 트랜잭션에 필요한 모든 정보가 개념적 스키마에 포함되어 있는지 확인할 수 있다.
  A. 3번
