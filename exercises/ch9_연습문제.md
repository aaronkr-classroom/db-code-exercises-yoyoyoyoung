# 9장 연습문제

## 개념퍼즐

**가로**

4. 하나의 업무 단위가 시간의 흐름에 따라 반복적으로 발생하거나 변경 또는 진행되는 과거 및 헌재 데이터 (이력데이터)
5. 데이터가 변경될 때마다 변경 전후으| 차이를 확인하기 위해 남기는 이럭 (변경이력)
6. 표준 용어를 구성히는 의미의 최소 단위인 단어를 표준화한 것 (표준단어)
8. DB에 저장된 데이터의 정확성과 일관성을 유지하기 위해서 DB가 향상 만족해야 히는 조건 (무결성제약조건)
9. 데이터 변경이 발생한 시작 시점부터 종료 시점까지 전체 시간을 관리하기 위해 남기는 이력 (선분이력)

**세로**

1. 데이터가 발생할 때마다 정보를 남겨서 관리하는 이럭 (발생이력)
2. DB에서 사용되는 데이터의 명칭， 정의， 형식， 규칙 등에 대한 전사적 표준을 수립하고 적용하는 과정 (데이터표준화)
3. 업무에 사용하는 용어에 대한 표준을 정의한 것 (표준용어)
7. 데이터 변경이 발생한 시점만 관리하기 위해 남기는 이력 (점이력)
10. 정보를 간단히 표현하기 위한 기호를 나타내는 개체 (코드개체)

## 연습문제

1. 개체 통합이 기능하지 않은 것은? (3)
  - 기본카가 동일한 개체 타입들
  - 기본키가 상호 식별자가 되는 개체 타입들
  - 모든 속성의 도메인이 동일한 개체 타엽들
  - 기본키와속성이 비슷한 개체 타입들

2. 이력 데이터에 대한 설명이 잘못된 것은? (2)
  - 이력 데이터란 하나의 업무 단위가 시간의 흐름에 따라 반복적으로 발생하거나 변경 또는 진행되는 과거 및 현재 데이터를 의미한다.
  - 변경 이력은 데이터가 변경될 때마다 변경 후의 상태를 확인해야 할 때 발생한다.
  - 이력 데이터의생성은발생 이력과변경 이력 진행 이력으로구분할수있다.
  - 발생 이력은데이터가발생할때마다이력 정보를남겨야할때 발생한다.

3. 개체 무결성 제약조건이 만족해야 하는 세 가지 규칙에 해당하지 않는 것은? (2)
  - 기본커를 구성하는 속성은 널 값을 가져서는 안 된다
  - 기본키는 데이터 보안이 요구되어야 한다.
  - 기본키는 유일성을 보장하는 최소한의 속성 집합이어야 한다.
  - 기본키는 개제를 유일하게 식별할수 있어야 한다.

4. 참조 무결성에 대한 설명이 잘못된 것은? (2)
  - 외래키는 피참조 릴레이션에 존재하는 기본키 값을 가져야 한다.
  - 외래키는 널 값을 가질 수 없다.
  - 참조 무결성은 외래키를 정의하면 묵시적으로 정의된다.
  - 참조 무결성의 세부 엽무 규칙을 정의할수 있다.

5. 참조 무결성의 세부 엽무 규칙에 대한 설명이 잘못된 것은? (3)
  - 널 (Set Null): 부모 인스턴스를삭제/수정하면， 대응하는 자식 인스턴스의 외래키를 널 값으로 설정한다.
  - 연쇄 (Cascade): 부모 인스턴스를 삭제/수정하면 대응히는 자식 인스턴스도 모두 자동으로 삭제/수정한다.
  - 제한(Restrict) : 대응하는 자식 인스턴스가 없는 경우 부모 인스턴스의 삭제/수정이 불가능하다.
  - 기본(Set Default): 부모 인스턴스를 삭제/수정하면 대응하는 지식 인스턴스의 외래커를 기본(디폴트)값으로 설정한다.

6. 명시적으로 정의해야 하는 도메인 무결성에 해당혜 않는 것은? (2)
  - NULL 제약조건
  - CHECK 제약조건
  - UNIQUE 제약조건
  - DEFAULT 제약조건

7. 데이터 표준화의 일반 원칙이 잘못 설명된 것은? (2)
  - 하나의 개념에 대해 하나의 명칭만 허용한다
  - 서술적 정의만으로 의미 전달이 기능하도록 상세히 작성한다.
  - 데이터가 의미히는 범위와자격 요건을 제3자도 이해할수 있도록 정의한다.
  - 성격이 유사한 데이터는 도메인을 정의해서 데이터 형식을 통일한다.

8. 데이터 표준의 주요 대상에 해당하지 않는 것은? (4)
  - 표준 단어
  - 표준 도메인
  - 표준 코드
  - 표준 개체

9. 데이터 표준화 절차가 바르게 작성된 것은? (1)
  - 요구사항수집 데이터 표준 정의 데이터 표준확정 데이터 표준 관리
  - 요구사항수집 데이터 표준 정의 - 데이터 표준 관리 - 데이터 표준 확정
  - 데이터 표준 관리 요구사항 수집 - 데이터 표준 정의 - 데이터 표준 확정
  - 데이터 표준 관리 - 요구사항수집 - 데이터 표준 확정 - 데이터 표준 정의

11. 어떤 클럽의 회원이 각종 행사에 참석한 내역을 기록하기 위해 다음과 같은 ER 다이어그램을 작성하였다. 이 ER 다이어그램을 릴레이션 스키마로 변환하시오. (351쪽에 보세요) (회원 (id, 이름, 휴대폰), 참석내역 (id(PK, FK), 참석일자, 회비납부여부)

12. 크게 계획 단계， 준비 단계， 철거 단계， 건축 단계， 마무리 단계， 입주 단계로 진행되는 아파트 재건축 공사의 진행 상황을 관라하기 위한 ER 다이어그램을 작성하고， 릴례이션 스키마로 변환하시오. 단， 아파트는 건물번호， 이름 · 주소 · 시공사 · 세대수 · 입주시작일 속성을 갖고， 진행단계는 순서 · 주관사 · 연락처 · 시작일자 · 종료일자 속성을 갖는다. (아파트 (건물번호, 이름, 주소, 시공사, 세대수, 입주시작일), 진행단계 (건물번호(FK), 순서, 주관사, 연락처, 시작일자, 종료일자))
